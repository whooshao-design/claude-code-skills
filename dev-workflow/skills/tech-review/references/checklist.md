# 详细评审检查清单

## Quick 粒度（3 轮）

### v1: 系统边界 + 目标清晰度

**系统边界检查**：
- [ ] 方案范围是否明确（做什么、不做什么）
- [ ] 与现有系统的边界是否清晰
- [ ] 上下游依赖是否完整列出
- [ ] 依赖的可行性是否已确认

**目标清晰度检查**：
- [ ] 方案目标是否可度量
- [ ] 成功标准是否明确
- [ ] 目标是否可实现

---

### v2: 核心流程 + 异常路径

**核心流程检查**：
- [ ] 主流程是否完整
- [ ] 关键节点是否都有说明
- [ ] 输入输出是否清晰

**异常路径检查**：
- [ ] 主要异常场景是否覆盖
- [ ] 异常处理策略是否明确
- [ ] 降级方案是否合理

---

### v3: 收敛 + 风险声明

**收敛检查**：
- [ ] 是否达到收敛条件
- [ ] 新问题是否减少
- [ ] 建议是否可操作

**风险声明**：
- [ ] 主要风险是否已识别
- [ ] 风险等级是否合理
- [ ] 缓解措施是否可行

---

## 金融风控场景专项检查

适用于风控系统技术方案：

### 风控核心检查

| 检查项 | 说明 | 级别 |
|--------|------|------|
| 决策一致性 | 相同输入是否产生相同决策 | Blocker |
| 特征计算幂等 | 特征复用场景是否幂等 | Major |
| 规则热更新 | 规则变更是否影响决策一致性 | Blocker |
| 决策延迟 | RT 是否满足风控要求 | Major |
| 规则优先级 | 规则冲突处理策略 | Major |
| 特征穿透 | 特征来源是否透明可追溯 | Completeness |
| 决策解释 | 拒绝决策是否有合理解释 | Major |

---

### 额度/资金场景检查

| 检查项 | 说明 | 级别 |
|--------|------|------|
| 额度扣减原子性 | 并发扣减是否原子 | Blocker |
| 资金一致性 | 资金流转是否可追溯 | Blocker |
| 幂等 Key 设计 | 幂等 Key 是否覆盖所有写操作 | Major |
| 状态机完整性 | 状态流转是否覆盖所有场景 | Major |
| 异常补偿机制 | 失败后是否能正确补偿 | Major |
